我在写一个可视化的SSD垃圾回收（GC）模拟器，
【目录结构】
demo/
├── index.html          # 主入口（精简后的HTML）
├── styles.css          # 样式文件
├── ssd-gc-simulator.html # 原文件（保留备份）
└── js/
    ├── config.js       # 配置管理模块
    ├── state.js        # 状态管理模块
    ├── utils.js        # 工具函数模块
    ├── writeStrategy.js    # 写入策略模块
    ├── gcStrategy.js       # GC策略模块
    ├── renderer.js     # 渲染模块
    ├── app.js          # 前端交互模块
    └── main.js         # 入口组装模块

需求：
1. UI主界面下方新增一个统计面板（柱状图），横轴为“block write age”，坐标轴从左到右符合'block write age: Old'-->'block write age: Recent'的变化趋势。
纵轴为'Valid%'（即block有效页占比），其中图表背景是深色，'Valid%'柱状图为亮色。
其中“block write age”是指该block最近一次写入距离当前的时长。时长参考写入LBA的数量来统计。
2. 参考实现方式：每个物理block维护一个包含有效页数和'block write age'等参数的列表，统计面板根据列表数据进行动态刷新。

补充：
1. 统计面板的“block write age”统计粒度为单个物理block，而非物理super block；
2. 物理block进行了'写入'动作的的判定标准为：该block此刻写入了逻辑页，旧页被标记为无效页时并非'写入'动作。
3. 统计面板随LBA写入动态刷新，请注意GC时也存在LBA写入。
