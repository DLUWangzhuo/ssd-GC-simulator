# SSD垃圾回收（GC）可视化模拟器 - 规范文档

## 1. 项目概述

- **项目名称**: SSD GC Visual Simulator
- **项目类型**: 交互式教学模拟器（单页HTML应用）
- **核心功能**: 可视化演示SSD垃圾回收原理，支持单步操作和回退前进
- **目标用户**: 存储系统学习者、SSD原理教学

## 2. SSD物理结构

### 2.1 容量配置
| 参数 | 值 |
|------|-----|
| Die通道数 | 4 |
| 每Die的Block数 | 5 |
| 每Block的Page数 | 9 |
| Super Block组成 | 1 Super Block = 4 Blocks（每Die各1个） |
| Super Block总数 | 5 |
| OP预留空间 | 1 Super Block (1/5比例) |
| 可用Super Block数 | 4 |
| 物理总页数 | 4 Die × 5 Block × 9 Page = 180 Page |
| 可用页数 | 180 × 4/5 = 144 Page |
| OP页数 | 36 Page |

### 2.2 寻址方式
- **逻辑页地址（LPA）**: 0 ~ 143（用户可寻址的逻辑空间）
- **物理页地址（PPA）**: 由Die、Block、Page组成

## 4. 功能规范

### 4.1 写入操作
- **顺序写**: 从逻辑地址0开始，顺序写入LPA
- **随机写**: 随机生成LPA进行写入
- **写入粒度**:
  - 单页写入 (1 Page)
  - 单块写入 (1 Block = 9 Pages)
  - 单Super Block写入 (1 Super Block = 36 Pages)

### 4.2 GC触发机制
- **触发条件**: 写入时可用容量 < 阈值（默认20%）
- **GC阈值**: 可配置（10%-30%）
- **GC过程**:
  1. 标记源Super Block（选择无效页最多的）
  2. 迁移有效数据到新位置
  3. 擦除源Super Block
- **不包含**: 空闲GC策略

### 4.3 操作历史
- 记录所有写入和GC操作
- 支持操作回退（Undo）
- 支持操作前进（Redo）
- 历史步数限制: 100步

### 4.4 逻辑地址映射表
- 显示LPA到PPA的映射关系
- 支持按LPA排序和按PPA排序
- 实时更新

## 7. 实现说明

### 7.1 文件位置
- 模拟器: `demo/ssd-gc-simulator.html`

### 7.2 技术实现
- 纯前端实现（HTML + CSS + JavaScript）
- 无需服务器，可在浏览器中直接打开
- 使用 Google Fonts (JetBrains Mono, Noto Sans SC)

### 7.3 核心算法
- **GC Victim选择**: 选择无效页最多的Super Block作为回收目标
- **有效数据迁移**: 将选中的Super Block中的有效页迁移到空闲页
- **块擦除**: 重置整个Super Block为空页状态
- **地址映射**: 维护LPA→PPA和PPA→LPA双向映射表
